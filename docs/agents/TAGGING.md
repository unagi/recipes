# TAGGING.md - タグ設計ガイド

## 目的
- タグの重複・乱立を抑え、検索性を高める
- レシピ間の横断的な絞り込み軸を揃える

## タグの役割
- `category`: 主カテゴリ（1つのみ）
- `tags`: 横断軸（2〜4個を目安）

## タグ分類（採用カテゴリ）
- **品目**: 前菜/副菜、デザート、飲み物
- **主菜素材**: 肉料理、魚料理、その他の主菜
- **料理タイプ（包括）**: 麺料理、ごはん料理、スープ料理 など
- **主材料**: 豚バラ、鶏胸肉、豆腐、たら など
- **保存/用途**: 作り置き、保存食、仕込み
- **郷土料理**: 地域由来の料理に付与
- **ソース/タレ**: タレ、ドレッシング など

## 命名ルール
- 名詞で統一し、短く一意にする
- 表記は日本語（漢字/ひらがな/カタカナ）で統一する
- 同義語は作らず、既存の代表タグに寄せる
- 1レシピ内のタグ数は2〜4個を目安にする
- 料理ジャンル（和食/中華/イタリアン等）はタグにしない（categoryで管理）
- 調理法・加工はタグにしない
- 特徴/テイストはタグにしない
- 主菜素材タグは「主菜のみ」に付与する（肉の使用量が少ない料理は除外）

## 新規タグの追加条件
- 既存タグで代替できない
- 3レシピ以上に適用予定がある
- 追加時に本ドキュメントへ追記する

## 決定済みの統合・禁止ルール
### 料理ジャンル（タグ禁止）
- 和食/中華/イタリアン/タイ料理などのジャンルは **カテゴリで管理** し、タグに付けない

### 料理タイプ（包括タグのみ）
- 細分類のタグは付けず、包括的なタイプで統一する
  - 例: `パスタ`/`焼きそば`/`冷やし担々麺` → `麺料理`

### 品目タグ
- `前菜/副菜` / `デザート` / `飲み物` を必要に応じて付与する
- `スープ料理` は料理ごとに判断する

### 主菜素材タグ
- `肉料理` / `魚料理` / `その他の主菜` を主菜に付与する
- 肉の使用量が少ない料理（例: 焼きそば）は `肉料理` を付与しない

### 郷土料理
- **郷土料理に分類できるレシピは `郷土料理` タグを付ける**
- **地域タグは付けない**（例: `沖縄`, `北海道`, `富士宮`, `駒ヶ根` は禁止）
- 地域名が料理名に含まれる場合も `郷土料理` を付与する
  - 例: 「沖縄そば」「富士宮焼きそば」「駒ヶ根ソースカツ丼」「北海道スープカレー」

### ソース/タレ系
- `タレ` を代表タグとして採用する
- `ソース` / `漬けダレ` / `継ぎ足し` は **`タレ` に統合**
- `ドレッシング` はサラダ用途のため **独立タグとして維持**

### 保存・仕込み系
- `作り置き` / `保存食` / `仕込み` を使用する
- `冷凍保存可` は **`作り置き` に統合**（保存方法は本文で補足）

## 運用メモ
- 料理名タグは「料理タイプ」で代替できる場合は付けない
  - 例: `スープパスタ` は `麺料理` で代替
- 迷った場合は「検索軸として機能するか」を優先して判断する
